36     * instances with appropriate error codes and messages. The mapping follows these rules:
37     * - AbstractRequestException: passed through unchanged (already JSON-RPC compliant)
38     * - AuthenticationException: mapped to UnauthorizedException (401)

42     * - ValidationException: mapped to UnprocessableEntityException with validation details (422)
43     * - All other exceptions: mapped to InternalErrorException (500)
44     *
45     * @param  Throwable                $exception the exception to map to a JSON-RPC exception type
46     * @return AbstractRequestException the mapped JSON-RPC exception with appropriate error code and message
47     */
